CanvasGPT Architecture

Canvas LMS AI Assistant: Modules, Technologies, And Current State

This document visualizes how the Chrome extension is currently organized, what each module does, and how the project is moving toward a Canvas-grounded AI side panel.

Product Flow

Chrome

Side Panel

User opens the extension on a Canvas page and asks for context or course material.

Canvas

Detect Page

The extension checks domain, route, course ID, and Canvas DOM markers.

Runtime

Collect Context

Current page HTML/text or visible course materials are collected through the browser session.

TypeScript

Parse + Retrieve

Canvas content becomes structured docs, then locally ranked snippets.

Next

Prompt + Chat

Retrieved snippets will feed the AI prompt with source citations.

Runtime Extension Modules

manifest.json

Extension Definition

Declares Chrome Manifest V3 permissions, side panel, content scripts, background service worker, host permissions, and extension metadata.

background.js

Coordinator

Handles active tab lookup, side panel behavior, Canvas validation, content-script messages, current-page collection, and course material requests.

sidepanel/

User Interface

Displays Canvas status, domain settings, current page collection, and visible course material summaries.

Canvas-Specific Runtime Modules

canvas/detect.js

Canvas Detection

  • Checks allowed Canvas hostnames.
  • Parses route types: assignment, modules, page, syllabus, discussion, announcement, file, quiz, unknown.
  • Provides DOM fallback detection for custom domains.
settings/canvas-domains.js

Domain Settings

  • Stores configured Canvas domains in Chrome storage.
  • Validates school-specific hostnames.
  • Supports defaults like *.instructure.com.
canvas/collect/

Current Page Collection

  • Uses chrome.scripting.executeScript to capture active page context.
  • Returns URL, title, content type, visible text, and transient raw HTML metadata.
  • Clears raw HTML after parsing the current page context.
content/

Canvas Page Scripts

  • content.js extracts DOM-visible page data and verifies Canvas pages.
  • canvasApiClient.js reads visible course materials through the logged-in same-origin Canvas session.
  • No Canvas developer key is used.

Parser Layer

The parser layer is TypeScript under canvas/parse/. It converts Canvas HTML into normalized CanvasContextDoc objects.

base.ts

Shared Contract

Defines CanvasContextDoc, metadata types, route types, text normalization, URL helpers, source hashing, and fallback parsing.

assignment.ts

Assignment Parser

Extracts title, instructions, due dates, availability, points, submission type, linked files, module breadcrumb, and visible rubric text.

module.ts

Module Parser

Extracts module titles and visible module item links without fetching linked pages.

page.ts

Wiki Page Parser

Extracts page body, linked files, and iframe metadata.

syllabus.ts

Syllabus Parser

Extracts syllabus body and visible course summary rows.

sanitize.ts

Sanitization

Removes scripts, styles, hidden content, navigation, sidebars, and other non-content elements before parsing.

Retrieval Layer

The retrieval layer is TypeScript under canvas/retrieval/. It selects relevant snippets locally before any AI model receives Canvas context.

chunk.ts

Chunking

Splits documents into token-bounded chunks, preserves metadata, estimates tokens, keeps overlap, and avoids splitting table-like rows.

keyword.ts

Keyword Scoring

Scores exact phrase, title, metadata, and body text matches using local query tokenization.

intent.ts

Intent Boosting

Detects due-date, submission, reading, and policy queries. Boosts assignment, rubric, syllabus, module, page, and file chunks accordingly.

hybrid.ts

Bounded Retrieval

Combines keyword score, intent boost, current page/module boost, and recency boost. Returns max 8 chunks, max 6 sources, and token totals.

Current Status

Implemented

TypeScript retrieval layer with chunking, scoring, intent boosts, bounded output, and tests.

Next

Testing And Quality Gates

Parser Tests

tests/canvas/parse/parser.test.ts

Validates assignment, module, page, syllabus, and fallback parser behavior against redacted HTML fixtures.

Retrieval Tests

tests/canvas/retrieval/retrieval.test.ts

Validates chunk bounds, metadata preservation, table row handling, submit/rubric boosts, source limits, and token totals.

Recommended Next Engineering Step

Introduce a build step, likely with Vite or another lightweight bundler, so the TypeScript parser and retrieval layers can be imported by the Chrome extension runtime. After that, the current page flow can become:

Collect Page

Read active Canvas page through browser APIs.

Parse

Create CanvasContextDoc documents.

Chunk

Create bounded local snippets.

Retrieve

Select relevant snippets for the student question.

Prompt

Send only retrieved snippets to the selected model with citations.
